# 注解速查表

> 注解就是给代码贴标签，告诉 Spring/Java："这个类/方法/字段有特殊用途，请帮我自动处理。"

---

## 一、IoC 容器 — 管理 Bean 的注解

### @Component
- **大白话**："这是我自己写的类，Spring 请帮我创建和管理它"
- **贴在哪**：类上
- **效果**：Spring 扫描到后自动 new 一个实例放进容器，别人需要时直接给

### @Repository
- **大白话**："这是操作数据库的类（DAO 层）"
- **和 @Component 的区别**：本质一样，只是多了一个"我是数据库层"的标签。Spring 会额外把数据库异常翻译成 Spring 的异常

### @Service
- **大白话**："这是写业务逻辑的类"
- **和 @Component 的区别**：本质一样，标签不同，方便一眼看出这是业务层

### @Controller
- **大白话**："这是处理 HTTP 请求的类"
- **和 @Component 的区别**：本质一样，标签不同，MVC 中控制器专用

### @Configuration
- **大白话**："这是一个配置类，里面定义了一堆要交给 Spring 管理的对象"
- **配合**：和 `@Bean` 一起用

### @Bean
- **大白话**："这个方法返回的对象，交给 Spring 管理"
- **什么时候用**：这个类是第三方库的，你没法进源码加 `@Component`，就用 `@Bean` 手动注册
- **贴在哪**：方法上（方法必须在 `@Configuration` 类里）

### @Autowired
- **大白话**："Spring，帮我自动把这个字段/参数填上"
- **怎么找**：默认**按类型找**——参数是 UserRepository 类型，就去容器里找 UserRepository 的实例
- **贴在哪**：构造器、字段、Setter 方法

### @Resource
- **大白话**："帮我注入，但默认按**名字**找"
- **和 @Autowired 的区别**：@Autowired 按类型，@Resource 按名字。@Autowired 是 Spring 的，@Resource 是 Java 自带的

### @Scope
- **大白话**："这个 Bean 的作用范围是什么？全局唯一还是每次都新建？"
- **常用值**：
  - `singleton`（默认）= 全局只有一个实例，反复拿都是同一个对象
  - `prototype` = 每次拿都 new 一个新对象

### @PostConstruct
- **大白话**："Bean 创建完、依赖注入完成后，自动调用这个方法做初始化"
- **类比**：手机开机后的自启动程序

### @PreDestroy
- **大白话**："容器销毁前，自动调用这个方法做清理"
- **类比**：关机前自动保存数据

## 二、Spring MVC — Web 请求的注解

### @RestController
- **大白话**："这个类的每个方法返回值直接转成 JSON 发给浏览器，不跳页面"
- **本质**：`@Controller` + `@ResponseBody`

### @RequestMapping
- **大白话**："这个方法对应哪个 URL 路径"
- **可以限定**：路径、请求方式（GET/POST）、请求参数等

### @GetMapping
- **大白话**："只处理 GET 请求"
- **GET 是干啥的**：查询/读取数据。浏览器输入网址就是 GET

### @PostMapping
- **大白话**："只处理 POST 请求"
- **POST 是干啥的**：新增/提交数据。表单提交一般是 POST

### @PutMapping
- **大白话**："只处理 PUT 请求"
- **PUT 是干啥的**：更新/替换已有数据（整个替换，不是部分修改）

### @DeleteMapping
- **大白话**："只处理 DELETE 请求"
- **DELETE 是干啥的**：删除数据

### @PathVariable
- **大白话**："从 URL 路径里取值"
- **举例**：`/api/users/123` → `@PathVariable Long id` 拿到 123

### @RequestParam
- **大白话**："从 URL 问号后面取值"
- **举例**：`/api/users/search?name=张三` → `@RequestParam String name` 拿到 "张三"

### @RequestBody
- **大白话**："把请求体里的 JSON 自动转成 Java 对象"
- **举例**：请求体 `{"name":"张三","email":"zs@test.com"}` → 自动变成 User 对象

### @ResponseBody
- **大白话**："方法的返回值直接写进 HTTP 响应体，不经过视图解析器"
- **一般不需要单独用**：`@RestController` 已经包含它了

## 三、AOP — 切面编程的注解

### @Aspect
- **大白话**："我是一个切面类，里面写了拦截规则和拦截后要执行的代码"

### @Before
- **大白话**："在目标方法执行**之前**执行我"
- **类比**：进门之前先脱鞋

### @After
- **大白话**："在目标方法执行**之后**执行我，不管成功还是异常"
- **类比**：吃完饭后离开餐厅（和 Java 的 finally 一样）

### @AfterReturning
- **大白话**："目标方法**正常返回**后执行我，我能拿到返回值"
- **类比**：吃完饭拿到账单

### @AfterThrowing
- **大白话**："目标方法**抛出异常**时执行我，我能拿到异常信息"
- **类比**：菜里有虫子，投诉

### @Around
- **大白话**："我把目标方法包围了，前后都能做事，还能决定原方法执行不执行"
- **最强大**：可以改入参、改返回值、甚至不调原方法（比如缓存命中跳过查询）

### @Transactional
- **大白话**："这个方法里的数据库操作，要么全成功，要么全回滚"
- **原理**：AOP 代理拦截 → 方法前开启事务 → 方法成功提交 / 异常回滚
- **失效场景**：同类方法调用不走代理、非 public、异常被 catch 吃掉、数据库引擎不支持事务

## 四、Spring Boot 启动注解

### @SpringBootApplication
- **大白话**："一键启动 Spring Boot，等于同时开启三件事"
- **包含**：
  - `@Configuration` = 允许定义 Bean
  - `@EnableAutoConfiguration` = 自动配置（根据你引入的 jar 包智能装配）
  - `@ComponentScan` = 扫描当前包下的所有 Bean

### @EnableAutoConfiguration
- **大白话**："根据 classpath 里的 jar 包自动配置项目"
- **举例**：你引入了 `spring-boot-starter-web` → 自动配好 Tomcat、DispatcherServlet、Jackson

### @Value
- **大白话**："把配置文件里的值读进来"
- **举例**：`@Value("${server.port}")` → 读取 application.yml 里的 server.port

## 五、Lombok — 自动生成代码的注解

### @Getter
- **大白话**："自动给所有字段生成 getter 方法"

### @Setter
- **大白话**："自动给所有字段生成 setter 方法"

### @NoArgsConstructor
- **大白话**："自动生成无参构造器"

### @AllArgsConstructor
- **大白话**："自动生成包含所有字段的构造器"

### @RequiredArgsConstructor
- **大白话**："自动生成构造器，但只包含被 `final` 或 `@NonNull` 标记的字段"
- **Spring 配合**：Service 中 `private final XxxRepository` + 这个注解 = 不需要手写构造器也能注入

### @Data
- **大白话**："一次性生成 getter + setter + toString + equals + hashCode"
- **等于**：`@Getter` + `@Setter` + `@ToString` + `@EqualsAndHashCode`

### @Slf4j
- **大白话**："自动创建一个 log 变量，可以直接写 `log.info(...)`"

## 六、Java 内置注解

### @Override
- **大白话**："我这个方法是重写父类的，请编译器帮我检查写对了没"
- **举例**：如果父类方法叫 `toString`，你写成了 `tostring`，编译器会报错

### @Deprecated
- **大白话**："这个方法/类已经过时了，不建议用，请用新的"
- **效果**：调用时 IDE 会划删除线

### @SuppressWarnings("all")
- **大白话**："关闭编译器的警告提示"

### @FunctionalInterface
- **大白话**："这是一个函数式接口（只有一个抽象方法），可以用 Lambda 表达式"

---

## 七、JUnit 测试注解

### @Test
- **大白话**："这是一个测试方法，直接运行我来验证代码对不对"

### @BeforeEach
- **大白话**："每个 @Test 方法运行前，先运行我（做准备工作）"
- **JUnit 4 叫**：`@Before`

### @AfterEach
- **大白话**："每个 @Test 方法运行后，运行我清理现场"
- **JUnit 4 叫**：`@After`

### @DisplayName
- **大白话**："给测试起个中文名字，测试报告里显示"

### @ParameterizedTest
- **大白话**："这个测试要用多组数据跑多次"

### @Mock
- **大白话**："创建一个假对象（Mockito），模拟行为但不真调用"

### @InjectMocks
- **大白话**："把上面那些 @Mock 创建的对象自动注入到我要测的类里"

---

## 八、Jackson JSON 注解

### @JsonFormat
- **大白话**："日期/时间按指定格式序列化"
- **举例**：`@JsonFormat(pattern = "yyyy-MM-dd")` → 日期转 JSON 时显示为 "2026-06-01"

### @JsonSerialize
- **大白话**："用指定的序列化器处理这个字段"
- **常用**：`@JsonSerialize(using = ToStringSerializer.class)` — 把 Long 转成 String，防止前端丢失精度

### @JsonIgnore
- **大白话**："把这个字段排除，序列化/反序列化时不处理"

### @JsonProperty
- **大白话**："JSON 中的字段名和 Java 字段名不一样时，指定映射关系"

---

## 九、Servlet 原生注解

### @WebServlet
- **大白话**："把这个普通 Java 类变成能处理 HTTP 请求的 Servlet"
- **贴在哪**：类上
- **举例**：`@WebServlet("/login")` → 访问 /login 就被这个方法处理

---

## 十、安全控制注解

### @PreAuthorize
- **大白话**："执行方法前先检查你有没有权限"
- **举例**：`@PreAuthorize("hasRole('ADMIN')")` → 只有管理员角色才能调这个方法

---

## 总结：注解的本质

```
注解 = 代码上的标签/便签

你把标签贴在类/方法/字段上
    ↓
Spring（或编译器/框架）扫描到标签
    ↓
自动执行对应的处理逻辑

你贴 @Service → Spring 帮你 new 好放进容器
你贴 @Autowired → Spring 帮你把依赖填进去
你贴 @Transactional → Spring 帮你管理事务
你贴 @Override → 编译器帮你检查拼写
你贴 @Test → JUnit 帮你跑测试
```

> 记住：**注解本身不干活，它只是个标记。真正干活的是扫描并处理这些注解的框架。**
