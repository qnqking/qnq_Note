# Java 位运算与原码、反码、补码

## 一、原码、反码、补码

计算机中**有符号整数**使用补码存储。以下以 8 位为例。

### 三种表示法

| 表示法 | 规则 | 用途 |
|--------|------|------|
| **原码** | 最高位符号位（0正1负），其余位表示数值 | 人直接可读，计算机不用 |
| **反码** | 正数=原码；负数=符号位不变，其余取反 | 补码的中间态 |
| **补码** | 正数=原码；负数=反码+1 | **计算机实际存储** |

### 示例：+5 和 -5（8位）

```
十进制   原码         反码         补码
+5      0000 0101    0000 0101    0000 0101
-5      1000 0101    1111 1010    1111 1011
```

### 为什么用补码？

1. **0 的唯一表示**：原码有 `+0(0000 0000)` 和 `-0(1000 0000)`，补码只有 `0000 0000`
2. **减法变加法**：`5 - 3 = 5 + (-3)`，CPU 只需要加法器即可完成加减
3. **多表示一个负数**：8位补码范围 `-128 ~ 127`，比原码多一个 `-128`

### 补码转十进制口诀

> 最高位是 1（负数）→ 取反加一，算数值，加负号

```
补码 1111 1011 → 十进制？
最高位=1，是负数：
取反 → 0000 0100
加1  → 0000 0101 = 5
结果：-5
```

## 二、位运算符

位运算直接操作二进制位，**速度极快**。

| 运算符 | 名称 | 规则 |
|--------|------|------|
| `&` | 按位与 | 全1则1 |
| `\|` | 按位或 | 有1则1 |
| `^` | 按位异或 | 不同则1 |
| `~` | 按位取反 | 0变1，1变0 |
| `<<` | 左移 | 高位舍弃，低位补0 |
| `>>` | 算术右移 | 低位舍弃，高位补**符号位** |
| `>>>` | 逻辑右移 | 低位舍弃，高位补**0** |

### 运算示例

```java
int a = 0b1100;  // 12
int b = 0b1010;  // 10

a & b   // 0b1000 = 8     全1才1
a | b   // 0b1110 = 14    有1就1
a ^ b   // 0b0110 = 6     不同才1
~a      // 取反 = -13     补码层面全部翻转
```

### 按位取反 ~n = -(n+1) 原理

```
~5  = -6：
  5 的补码：0000 ... 0101
  ~5      ：1111 ... 1010  (这是一个补码，最高位=1)
  对这个补码取反加一 → 0000 ... 0110 = 6，加负号 → -6
```

> 口诀：`~n == -n - 1`

### 位移运算

```java
int n = 0b0000_1100;  // 12

n << 2    // 0b0011_0000 = 48    左移 2 位 = 乘 4
n >> 2    // 0b0000_0011 = 3     算术右移 2 位 = 除 4
n >>> 2   // 同上（正数 >> 和 >>> 一样）

// 负数的情况
int m = -8;  // 补码：1111 ... 1000
m >> 2      // 补码：1111 ... 1110 = -2    左边补符号位1
m >>> 2     // 补码：0011 ... 1110 = 大正数  左边补0，变成正数！
```

### 位移替代乘除

```
n <<  1  →  n × 2
n <<  3  →  n × 8
n >>  2  →  n ÷ 4   (向下取整)
n >>> 1  →  无符号 n ÷ 2
```

## 三、经典操作技巧

```java
// 1. 判断奇偶
(n & 1) == 1  // 奇数
(n & 1) == 0  // 偶数

// 2. 交换两数（不用第三个变量）
a ^= b;
b ^= a;
a ^= b;

// 3. 将第 k 位置 1
n | (1 << k)

// 4. 将第 k 位置 0
n & ~(1 << k)

// 5. 取第 k 位的值
(n >> k) & 1

// 6. 消掉最低位的 1
n & (n - 1)          // 用于计算二进制中 1 的个数、判断 2 的幂

// 7. 只保留最低位的 1
n & -n                // lowbit，树状数组核心

// 8. 判断是否为 2 的幂
(n & (n - 1)) == 0 && n > 0
```

## 四、字节序补充（Big-Endian vs Little-Endian）

| 方式 | 规则 | 代表 |
|------|------|------|
| 大端 (Big-Endian) | 高字节存低地址 | Java JVM、网络协议 |
| 小端 (Little-Endian) | 低字节存低地址 | x86 CPU |

```
32位整数 0x12345678 在内存中的存储：

大端：12 34 56 78  (从左到右)
小端：78 56 34 12  (从右到左)

Java 是大端序，与 x86 CPU 相反，但 JVM 自动处理，程序员无感。
```
