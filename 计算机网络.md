# 计算机网络

## OSI 七层架构

| 层级 | 说明 |
|------|------|
| 物理层（Physical） | 网线/WiFi/路由器/交换机，传输电信号（高电压=1，低电压=0） |
| 数据链路层（Data Link） | 以太网协议（Ethernet），数据帧（Frame）= head（MAC地址） + data |
| 网络层（Network） | 划分子网，IP 协议，IP 地址 32 位（前 24 位子网，后 8 位主机） |
| 传输层（Transport） | 端口到端口的通讯（TCP / UDP） |
| 会话层（Session） | — |
| 表示层（Presentation） | — |
| 应用层（Application） | HTTP / HTTPS / FTP / AMQP / POP3…… |

> 下 3 层跟硬件有关，上 3 层跟软件有关，中间跟传输有关

## TCP/IP 五层架构

物理层 → 数据链路层 → 网络层 → 传输层 → 应用层

---

## TCP 协议

面向连接、可靠、三次握手、四次挥手。类似于打电话。

### 三次握手（Three-way Handshake）

客户端请求 → 服务器响应 → 客户端确认

目的：防止历史连接请求造成资源浪费。

### 四次挥手（Four-way Wavehand）

客户端请求断开 → 服务器收到 → 服务器清除消息 → 正式断开

原因：全双工通信，每个方向需独立关闭。

---

## UDP 协议

无连接、不可靠、速度快。类似于寄包裹/发短信。

---

## ARP 协议

根据 IP 地址获取目标设备的 MAC 地址。

MAC 地址：48 位，前 24 位=厂家，后 24 位=网卡序号。

---

## IP 地址 & 子网掩码

IP 地址与子网掩码按位与（AND）运算，结果相同 = 同一子网。

## HTTP 协议

### 请求协议（Request）

- 请求行：提交方法、请求路径、协议版本
- 请求头：浏览器给后端服务器的信息
- 空行：分隔
- 请求体：携带的数据

### 响应协议（Response）

- 响应行：协议版本、状态码、描述
- 响应头：后端返回的关键信息
- 空行：分隔
- 响应体：返回前端数据或页面

### 状态码

| 范围 | 含义 |
|------|------|
| 1xx | 后端收到请求，但未处理完毕 |
| 2xx | 处理成功 |
| 3xx | 重定向 |
| 4xx | 前端错误（如 404 资源不存在） |
| 5xx | 后端错误（服务器内部错误） |

### GET vs POST

1. GET 参数从 URL 传输 / POST 通过请求体
2. GET 只能传输文本 / POST 可传输二进制
3. GET 受 URL 长度限制 / POST 理论上无限制
4. GET 可缓存 / POST 不会被缓存 → POST 更安全
